# SKILL.md - 无锡市公共资源交易中心招标信息抓取技能

## 简介

本技能从**无锡市公共资源交易中心**（ggzy.wuxi.gov.cn）抓取招标公告信息，提取关键字段，下载 PDF 招标文件，同步至腾讯文档。

适用于招标信息监控、资质匹配筛选、邮件推送等场景。

---

## 平台配置（必填 - 明天提供）

> ⚠️ 网络封锁环境无法自动探测，需手动填写以下配置后运行。

### 需要提供的信息

请在浏览器中打开无锡公共资源交易中心，找到招标公告列表页，然后：

**1. 列表页 URL**
- 打开 https://ggzy.wuxi.gov.cn → 招标公告 → 施工
- 复制浏览器地址栏 URL，填入 `LIST_API_URL`（去掉 page= 参数部分）

**2. AJAX 详情接口**
- 按 F12 → Network → 点击任意一条公告
- 找一个返回 JSON 的 AJAX 请求（包含 `infoid` 和 `siteGuid` 参数）
- 复制请求 URL 填入 `DETAIL_AJAX_URL`，复制 `siteGuid` 值填入 `SITE_GUID`

**3. 腾讯文档**
- 新建一个空白腾讯文档表格
- 填入 `TENCENT_FILE_ID`（从文档 URL 中提取 `/sheet/Dxxxxxxx` 后的部分）
- 填入 `TENCENT_SHEET_ID`（通常为子表 ID，如 `tab-xxx`）

## 持久积累模式

脚本每次运行自动维护一份**持续积累 Excel**（`outputs/无锡招标_持续积累.xlsx`），逻辑：

1. **读取**本地已有记录（按 section_id 去重）
2. **合并**本次抓取的新条目
3. **清理**发布时间距今 >7 天的过期条目 + 对应本地 PDF 文件夹
4. **写入**同一份持久 Excel（固定路径，不按日期命名）
5. **同步**腾讯文档时基于这份完整记录全量重写

```
outputs/
├── 无锡招标_持续积累.xlsx      ← 持续积累Excel（固定路径）
├── 20260409_xxxxxxxx/         ← PDF 文件（每个项目一个文件夹）
└── 20260408_yyyyyyyy/
    └── 招标文件正文.pdf
```

## 运行方式

```bash
cd /root/.openclaw/workspace/skills/wuxi-bid-search

# 模拟运行（预览合并结果，不实际写入）
python3 scripts/scrape_v2.py --sync --dry-run
```

## PDF 字段提取规则

### 评标基准价计算方法（2.3.2）
从"评标基准价计算方法"一节提取：
- **计算方法编号**：从 `☑...方法X` 选择语句提取
- **方法名称**：在同节 `方法X：ABC合成法` 描述中提取
- **K 取值范围**：在 `K 值的取值范围为：...` 段落提取 % 值
- **下浮率Δ**：在 `本次招标项目下浮率Δ分类为` 表格中查找该工程类别对应的取值范围

格式：`ABC合成法，K（95%-98%），下浮率Δ（6%-12%）`

### 综合评估法评分组成（2.3.4）
仅综合评估法项目：
- **施工组织设计**：评分因素表格中 ☑ 行末尾的 X 分累加（跳过答辩/□其他/票决法）
- **答辩**：从 `总分X 分` 提取
- **报价**：从 `投标报价评审...X 分` 提取
- **信用分**：从 `×N/100` 提取

### 评标入围方法（第三章）
在"评标入围"正文区域搜索 `☑/□方法X`，提取选中方法名称及"不少于 X 家"

## 九、自动迭代机制

每次 `--sync` 运行结束后自动进行**字段质量复盘**：

1. 对本次抓取的每条记录，从 PDF 原文核查关键字段（budget、评标办法、资格审查等 11 项）
2. 对比提取值与 PDF 原文，标记 ✅/⚠️/❌/N/A
3. 统计本次质量评分并写入 `scripts/changelog.md`

复盘结果在每次运行末尾输出：
```
[字段质量复盘] ⚠️ 有字段未提取 | 161/176
```

**查看 changelog**：`cat scripts/changelog.md`

## 更新日志

### 2026-04-08 PDF 字段提取重构

适配无锡 2025 版标准施工招标文件（省标）格式，主要修复：

| 问题 | 原因 | 修复方式 |
|:--|:--|:--|
| 项目名称仅得"一期(C10)施工" | 2.1节只有标段名，无项目名 | 从招标公告标题 `东璟家园C7、C10装饰工程（项目名称）一期(C10)施工（标段名称）` 提取完整项目名 |
| 招标人含"建设资金来..." | `招标人为`后面内容未截断 | 前附表精确提取，或截断到第一个中文逗号 |
| 招标代理含"（盖单位公章）：" | 同上 | 同上 |
| 控制价/工期/投标有效期取不到 | 字段名/格式差异（2.6节 vs 最高投标限价、2.9节 vs 前附表） | 优先精确匹配 `工程合同估算价（万元）` / `2.9 工期` / `天（从投标截止之日起算）` |
| 评标基准价计算方法空白 | 正则未匹配 `K 值的 取值范围`（中间有空格） | 修复正则 `K\s+值.{0,5}取值范围`，支持数字间有空格 |
| 评标基准价只有下浮率，无K值 | 同上 + 下浮率范围正则 `[\d%，.%～-]+` 漏了顿号`、` | 同上 + 范围正则加 `、` |
| 资质要求/项目负责人资质超长 | 正则 `3\.1 投标人资质类别和等级[：:]\s*([^\n]+)` 含多行 | 改为只取方括号内内容 `\[([^\]]+)\]` |
| agent_contact 字段未定义 | 遗漏 | 从前附表 `1.1.3 招标代理机构` 提取电话 |

**关键技术点：**
- 无锡 PDF 评标基准价计算有三种方法（方法一A×K法、方法二A×K1×Q1+B×K2×Q2、方法三ABC合成法），通过 `☑方法X` 标记选中
- `K值的取值范围` 在三种方法中数值相同（95%~98%），但 `下浮率Δ` 和 `Q1` 的范围因工程类别不同而有差异
- `投标有效期` 格式为 `90 天（从投标截止之日起算）`，通过正则 `(\d+)\s*天（从投标截止之日起算）` 精确提取
