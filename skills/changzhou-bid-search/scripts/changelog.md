### 2026-04-10 问题修复记录

**现象**：本地 Excel 从 3 条变成 2 条，B320404（广仁路/浦锦路）详细字段（控制价、工期等）丢失。

**根因**：4 个 bug 同时发作：

1. **`merge_and_cleanup` key 不一致**：`hist_map` 用 `'标段编号'` 作为 key，但遍历 `new_items` 时用 `it.get('section_id')`。历史记录读回后 key 是 `'标段编号'`，merge 时 new_item 找不到匹配项导致当作新增处理（key=`NEW_uuid`）。

2. **`make_xlsx` 和 fallback 路径 key 错误**：写入时用 `'budget'`, `'duration'` 等英文 key，但历史 Excel 数据只有 `'控制价(万元)'`, `'工期'` 等中文列名，导致历史数据字段全被覆盖成空。

3. **`read_persistent_excel` 使用 `data_only=True`**：当 Excel 被 pandas 写入时，读回 None 值而非实际数据。

4. **`_fv` helper 缺失**：`make_xlsx` 和 fallback 路径没有统一处理英文 key（`'budget'`）和中文 key（`'控制价(万元)'`）的兼容性。

**修复**：
- `merge_and_cleanup`：在合并后整体标准化，将 `'标段编号'` → `'section_id'`, `'项目名称'` → `'project_name'`, `'发布日期'` → `'date'`
- `merge_and_cleanup`：修复 `if v` 判断，阻止 `None` 值覆盖已有的有效值
- `make_xlsx`：添加 `_fv(d, 'budget', '控制价(万元)')` 等兼容中文 key
- `write_persistent_excel` fallback 路径：添加 `_fv` helper 兼容中文 key
- `read_persistent_excel`：移除 `data_only=True`，并对空单元格填充 `''`

**当前状态**：2 条记录（常州旅游商贸 04-09 + 广仁路/浦锦路 04-08），全部字段完整。

**教训**：同一字段在 HTML（英文 key）、Excel（中文列名）、腾讯文档（中文列名）之间有多套命名，必须用 `_fv` helper 统一兼容。
